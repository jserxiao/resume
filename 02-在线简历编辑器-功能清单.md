## 四、块系统（核心）

### 4.1 基本概念

编辑器没有预定义的信息块类型，而是提供一套通用的**块模板 + 字段系统**。
用户先定义块模板（即一组字段的布局），再用模板创建具体的块实例填写内容。

```
块模板 = { 模板名称, 字段列表(字段名 + 字段类型 + 排序 + 默认值) }
块实例 = 块模板 + { 字段值 }
```

## 五、配色系统

### 5.1 基本概念

简历配色不是"选一个颜色"，而是一套完整的**色彩方案（Color Scheme）**。
每个色彩方案由以下角色构成：

```
色彩方案 = {
  主色(Primary)     — 标题、块标题、分割线、图标（1 个主色调）
  辅色(Secondary)   — 标签背景、高亮区域、装饰元素（1 个辅色调）
  背景色(Background)— 整页底色、块底色（1-2 个色值）
  文字色(Text)      — 正文、副标题、标注文字（2-3 个灰度层级）
  强调色(Accent)    — 链接、按钮、评分色、标记色（1 个鲜明色）
}
```

### 5.3 程序化生成色彩方案

编辑器内置一个**色彩方案生成引擎**，基于色相、饱和度、明度的算法规则自动生成协调方案。

| # | 功能 | 优先级 | 说明 |
|---|------|--------|------|

## 六、布局系统

### 6.1 基本概念

布局是指简历的整体骨架结构，在用户创建简历时先选布局方案。
布局决定块的物理排布方式，不涉及具体内容和颜色。

```
布局方案 = { 分栏方式, 页头样式, 信息密度, 页面尺寸, 分页策略 }
```

### 6.2 创建简历时选择布局

| 6.2.4 | 选择布局后进入编辑器，布局不可逆（或提示"切换布局可能导致排版混乱"） | P1 | 保持认知一致性 |
